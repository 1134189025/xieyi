# 审查修复与前端汉化计划

## 当前阶段进度

- 2026-06-01：阶段 1 至阶段 6 已完成，正在做最终验证。

## 已完成内容

- 新增 Vitest 作为最小测试框架。
- 新增配置校验、订单状态、错误脱敏、Stripe 超时参数和前端 API 分离的红灯测试。
- 已实现启动配置强校验、当前用户鉴权校验、登录限流和 `/api/auth/me`。
- 已新增 `CREATING_PAYMENT` 状态和 Prisma migration，订单创建、完成、取消改为条件状态写入。
- 已为 ChatGPT 与 Stripe 请求增加 30 秒超时边界，并对公开错误做脱敏。
- 已拆分公开/鉴权 API，请求 401/403 时清理前端鉴权状态。
- 已直接汉化前端主要用户可见文案，并保留 Pix、ChatGPT、Stripe 等专有名词。
- 已删除根目录重复核心源码，根 `cli.ts` 改为兼容入口，权威实现保留在 `packages/core`。
- 已升级 `bcrypt` 到 `^6.0.0`，生产依赖审计不再报告 `tar` 漏洞链。

## 关键决策

- 前端直接改成简体中文，不引入 i18n 框架。
- 失败订单继续消耗兑换码，保持既有业务假设。
- `CREATING_PAYMENT` 只作为客户侧处理中状态，不进入工人待处理列表。
- Recharts 2.x 维护风险本轮只登记，不升级。

## 必需笔记

- 本轮会提交 Prisma migration，因为订单状态枚举需要增加 `CREATING_PAYMENT`。
- 外部错误只允许写入安全中文文案，原始错误只能进服务端日志。
- 复制 `.env.example` 后必须替换所有 `REPLACE_WITH_*` 值，否则服务端会拒绝启动。

## 后续注意事项

- 视觉检查没有可用的 in-app browser 控制工具时，以 `npm run build` 和文案扫描作为替代验证，并在最终说明中注明。
